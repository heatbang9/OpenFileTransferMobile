# 모바일-to-모바일 추가 피쳐

모바일-to-모바일은 모바일 앱이 클라이언트와 임시 서버 롤을 모두 갖는 흐름입니다. 우선 앱 활성 상태 중심의 단기 수신 모드로 안정화하고, Android는 foreground service 기반 장시간 수신으로 확장합니다.

## 구현 완료

- 임시 모바일 서버: `10분 열기`로 gRPC 서버, SSDP responder, HTTP descriptor를 시작합니다.
- 모바일 발견: 다른 모바일 앱의 `서버 찾기`에서 임시 서버를 OpenFileTransfer 서버로 찾습니다.
- 파일 송신: 기존 `SendFile` client-streaming으로 모바일에서 모바일로 파일을 보냅니다.
- 파일 조회/수신: 임시 모바일 서버도 `ListFiles`와 `ReceiveFile`을 제공해 수신함 파일을 다시 내려받을 수 있습니다.
- 신뢰 목록: 상대 UUID, 이름, 항상 허용 여부, 최근 발견 시각, 수신 횟수를 저장합니다.
- 승인 UX: 미승인 모바일은 `이번만 허용`, `항상 허용`, `거부` 팝업을 통과해야 파일을 주고받습니다.
- 진행률: 모바일-to-모바일 송수신 진행률을 앱 UI와 Android foreground service 알림에 반영합니다.
- 모바일 1:N 전송: 발견된 여러 PC/모바일 서버를 체크하고 같은 파일을 순차 전송합니다.
- 대상별 결과: 1:N 전송 후 각 대상의 성공/실패 결과를 전송 패널에 표시합니다.
- 모바일/PC 구분: 탐색 목록에서 모바일 임시 수신 서버와 PC 서버를 아이콘/라벨로 구분합니다.
- Android 수신 대기 알림: 모바일 수신 모드가 켜져 있는 동안 foreground service 알림을 유지하고, 송수신 완료 후 다시 수신 대기 알림으로 돌아갑니다.

## 다음 구현 후보

- 미승인 수신 요청을 앱이 전면에 없을 때 foreground notification action으로 허용/거부합니다.
- 1:N 전송 큐에 재시도, 취소, 대상별 진행률 고정 표시를 추가합니다.
- 신뢰 목록에서 디바이스 삭제, 이름 변경, 마지막 전송 시간 표시를 추가합니다.
- iOS는 장시간 백그라운드 서버 대기 제약이 크므로 전면/제한 백그라운드 수신을 우선 검증합니다.
